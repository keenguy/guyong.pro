Git index notes

Remote repository 是存储在远程服务器（如github)上的项目版本。管理remote repo会涉及到上传（push）和下载（pull）数据 。

Showing or inspecting your remotes

git remote [-v] 查看已经关联的remote信息
git remote show <origin> 查看origin的详细信息，包括remote信息以及tracking branch信息，如：

$ git remote show origin
* remote origin
  URL: https://github.com/my-org/complex-project
  Fetch URL: https://github.com/my-org/complex-project
  Push  URL: https://github.com/my-org/complex-project
  HEAD branch: master
  Remote branches:
    master                           tracked
    dev-branch                       tracked
    markdown-strip                   tracked
    issue-43                         new (next fetch will store in remotes/origin)
    issue-45                         new (next fetch will store in remotes/origin)
    refs/remotes/origin/issue-11     stale (use 'git remote prune' to remove)
  Local branches configured for 'git pull':
    dev-branch merges with remote dev-branch
    master     merges with remote master
  Local refs configured for 'git push':
    dev-branch                     pushes to dev-branch                     (up to date)
    markdown-strip                 pushes to markdown-strip                 (up to date)
    master                         pushes to master                         (up to date)

This command shows which branch is automatically pushed to when you run git push while on certain branches. It also shows you which remote branches on the server you don’t yet have, which remote branches you have that have been removed from the server, and multiple local branches that are able to merge automatically with their remote-tracking branch when you run git pull.

Adding remote repository

git remote add [-t <branch>] [-m <master>] [-f] [--tags | --no-tags] [--mirror=<fetch|push>] <name> <url>

Pushing to Your Remotes

$ git push origin master 会把当前的master branch push到origin上去。若origin上有本地没有的数据（可能有他人更新过），则需要先fetch并merge后才能push.

Fetching and Pulling from Your Remotes

$ git fetch [remote-name]

该命令会把所有remote repo（假设叫origin)有而本地没有的数据下载（pull)下来，之后就可以查看或操作origin上的branch. 需要注意的是git fetch只会下载数据，不会进行任何merge操作，也不会改变本地已有的数据。
如果current branch 关联（track)了一个remote branch, 那么git pull命令会自动的fetch并merge那个remote branch到current branch上。

Removing and Renaming Remotes

git remote rename
git remote remove
